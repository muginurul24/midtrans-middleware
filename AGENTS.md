Saat membuat komponen, gunakan teks atau copywriting yang jelas, relevan, dan bermanfaat bagi client. Hindari penggunaan kalimat yang tidak memiliki nilai atau tidak membantu pengguna memahami fungsi komponen tersebut.

Silakan kerjakan file AI_IMPLEMENTATION_CHECKLIST.md. Jika ada task yang sudah selesai dikerjakan, ubah statusnya dan beri checklist agar AI agent lain, atau developer berikutnya, dapat memahami progres terakhir dan melanjutkan implementasi dengan langkah terbaik.

Jika membuka port untuk kebutuhan testing, pastikan port tersebut ditutup kembali setelah selesai digunakan.

Saat ini project masih dalam tahap pembangunan dan belum menggunakan data asli atau data production. Karena itu, tidak perlu ragu untuk melakukan perubahan yang diperlukan, seperti menghapus data, menjalankan migrate:fresh --seed, mengubah file .env, menambahkan key yang memang seharusnya ada di .env, atau menghapus file yang perlu diperbaiki.

Pastikan seluruh perubahan dilakukan dengan tujuan membuat project menjadi lebih clean, konsisten, dan siap untuk deployment ke VPS. Untuk styling dan komponen UI, gunakan shadcn agar tampilan lebih rapi, reusable, dan sesuai standar project.
